执行总结
随着健康科技和女性健康意识的提升，经期追踪应用日益普及。本报告聚焦于设计一套基于用户输入（月经开始日期+平均周期长）的经期自动分期和提示系统。系统首先要定义并识别月经期、卵泡期、排卵期、黄体期等生理阶段（图示见下方），并针对每个阶段提供基于证据的能量、情绪、运动和工作节奏建议。算法部分将根据用户最后一次来潮日和周期长度计算当前周期天数，并据此划分阶段；还需处理数据不完整或周期不规则等边界情形。我们还将调研现有主流经期应用的实现方法，比较其算法特点和隐私保护。
循证研究表明：卵泡期（月经开始后至排卵）通常能量水平逐步升高、心情改善，适合高强度运动和高效工作；排卵期（约周期中点）往往精力峰值、自信心和社交欲望高；黄体期后半段（尤其临近下次来潮前）则常见疲倦、易怒、情绪波动，此时应适度降低运动强度、强调放松【18†L614-L623】【16†L84-L90】。月经期初期由于血量流失和激素骤变，常见乏力、情绪低落，但伴随月经结束，能量会恢复【18†L586-L594】【18†L614-L623】。针对以上规律，本系统将提供个性化提示。例如，月经期建议“保持休息、轻度拉伸”；卵泡期鼓励“利用高能量窗口加强锻炼和计划重要任务”；黄体期则提醒“防范PMS症状，适量运动可缓解压力”。
隐私方面，月经数据属于敏感健康信息（欧盟GDPR视为特殊类数据【29†L181-L189】），系统应严格保护用户数据安全，如加密存储、本地优先、不向第三方出售等，以赢得用户信任并符合法规【29†L158-L163】【29†L181-L189】。接下来的报告内容包括：生理学背景与阶段定义、算法设计（含伪代码和边界处理）、各阶段指导文案模板、现有应用比较表以及实现注意事项。
生理学背景与周期阶段
女性月经周期是指从一次月经第一天开始到下一次月经开始前的全部过程。根据权威资料，正常月经周期长度约24~38天【1†L109-L118】【11†L43-L49】，平均28天。月经周期可分为卵泡期、排卵期、黄体期（图表参见下文）。其中，卵泡期从月经第1天（第一次有重度出血日）开始【1†L109-L118】；排卵期为卵子释放当天，持续约16–32小时【6†L123-L130】【11†L73-L79】；黄体期始于排卵后，固定约14天左右【8†L160-L164】【22†L695-L700】；月经期实际流血时间平均4–6天（可2–8天变动范围）【22†L695-L700】【11†L47-L49】。在医学上，卵泡期往往又细分为增殖期（子宫内膜增厚）和排卵前期，但对于周期追踪，可按（1）月经（出血期）、（2）卵泡期（含增殖期）、（3）排卵期、（4）黄体期（含分泌期）这四个阶段划分。
•	月经期（出血期）：周期第1天开始，一般持续4–7天【2†L197-L203】【22†L695-L700】。此时子宫内膜脱落并排出，经血约30ml（正常范围10–60ml）。
•	卵泡期（增殖期）：从月经结束后（也可含出血末期）起算，到排卵前夕结束。此期促卵泡激素（FSH）作用下卵泡发育成熟，雌激素水平升高，内膜增厚【11†L61-L70】【6†L115-L119】。卵泡期时长可变（约9~16天不等），临近绝经期变短【11†L68-L72】。
•	排卵期：通常发生在下次预期月经前约14天左右，持续约1天【6†L123-L130】【11†L73-L79】。成熟卵泡破裂释放卵子。围绕排卵点激素变化剧烈，是高生育力窗口期。
•	黄体期（分泌期）：从排卵后一天开始，约持续14天【8†L160-L164】【22†L695-L700】。此期黄体分泌大量孕酮和雌激素，维持内膜环境。若未受孕，黄体期末孕酮骤降，触发下次月经来潮。

阶段特征与指导建议（循证）
基于现有研究和专家指南，不同月经阶段的体能、情绪、运动适宜度和工作状态存在规律性变化。以下根据证据概述各阶段的典型表现和建议：
•	月经期（Menstruation）：血量流出和雌激素、孕酮骤降常导致乏力、肌肉酸痛、头痛等症状【18†L586-L594】；睡眠质量可能下降，容易出现情绪波动（烦躁、悲伤）【18†L595-L603】。建议：重视休息和营养，饮食均衡补充蛋白质和铁质【18†L599-L603】；进行轻度活动（如散步、瑜伽、拉伸）可缓解经痛和酸痛【18†L600-L603】；工作节奏可适当放缓，避免剧烈运动或重体力活。
•	卵泡期（Follicular）：激素水平逐渐回升（尤其雌激素增加）【16†L134-L142】。女性往往感觉精力充沛、心情愉快【18†L610-L618】【16†L84-L90】。Kaiser医生建议：此阶段“相对平静，精力和动力上升，适合高强度锻炼”【18†L610-L618】。证据：科研发现，半蹲力量在卵泡后期和排卵期间达到峰值，体力表现最佳【16†L84-L90】。建议：抓住高能量时段安排高强度运动或重要任务，例如力量训练、有氧运动；职业工作中可安排决策、演讲等需要高专注的事项。注意保持健康饮食及睡眠，以支持上升的体能需求。
•	排卵期（Ovulation）：一般在周期中期，卵子释放时。特点：激素峰值（LH峰值）阶段，体温略升高【8†L160-L164】。许多女性在此时注意力集中，信心和社交欲望增加【18†L637-L647】；体力上也通常处于高点。建议：利用高峰体能继续中高强度训练（短跑、间歇训练等）；此时工作精力旺盛，可处理较高负荷的工作。注意饮食和补水，排卵时可能出现轻微一侧腹痛（经间痛），可通过热敷等缓解。
•	黄体期（Luteal）：分为前黄体期（排卵后~黄体中期）和后黄体期（接近下次经期前）。初期体能仍较好，但后期（尤其最后一周）孕酮升高常引发经前综合征（PMS）【18†L669-L673】。表现：可能出现疲惫、乳房胀痛、腹胀、失眠、头痛、情绪低落或焦虑等【18†L659-L667】【18†L669-L673】。科研也发现：黄体晚期女性自觉精力下降、抑郁评分上升【16†L84-L90】。建议：适度运动仍有助于缓解PMS（例如瑜伽、快走或轻度有氧【18†L675-L679】），但可降低强度并增加放松活动；注意休息和深度睡眠，避免高压工作。合理饮食，控制甜食摄入（避免血糖骤升再骤降造成情绪波动）【18†L681-L683】，可少量摄入富含抗氧化物的黑巧克力等代替品。密切关注心情，如出现严重焦虑或抑郁，应咨询医生。
以上阶段特点参考了权威来源和研究。一项运动生理研究指出：“在卵泡后期和排卵期间半蹲力量达到峰值，在黄体后期最低；黄体后期疲劳和抑郁评分上升，活力降低”【16†L84-L90】。Kaiser Permanente健康教育也总结了各期常见症状和活动建议【18†L584-L594】【18†L610-L618】。可据此设计阶段性提示，帮助用户同步自身生活节奏与生理周期。
阶段计算算法与伪代码
算法目标：基于用户最后一次月经开始日（last_period_date）和平均周期长度cycle_length（天），计算当前为周期第几天day_in_cycle，并据此确定所处阶段。常用假设：黄体期固定约14天【22†L695-L700】；月经期长度默认取5天（可根据用户反馈调整）。
基本步骤如下：首先计算从last_period_date起至“今天”总天数，并取周期长度cycle_length的模（余），得到当前周期中的天数day_in_cycle。然后根据day_in_cycle与各阶段边界判断阶段。示例伪代码：
function getCyclePhase(last_period_date, cycle_length):
    if last_period_date is null OR cycle_length is null:
        return "未知阶段"  # 无有效输入

    today = current_date()
    days_since_start = (today - last_period_date).days
    if days_since_start < 0:
        return "未来日期错误"  # 起始日期在未来
    day_in_cycle = (days_since_start % cycle_length) + 1  # 1-based

    # 阶段边界假设
    period_length = 5  # 默认经期长度
    luteal_length = 14  # 黄体期长度
    follicular_length = cycle_length - luteal_length  # 卵泡期总长（含月经期和增殖期）

    # 决定阶段
    if day_in_cycle <= period_length:
        phase = "经期"
    elif day_in_cycle <= follicular_length:
        phase = "卵泡期"
    elif day_in_cycle == follicular_length + 1:
        phase = "排卵期"
    else:
        phase = "黄体期"

    return phase
边界情况处理：
•	周期长度过小：若cycle_length < period_length + luteal_length（例如<19天），则无效（提示用户重设合理值）。
•	经期长度未输入：上述算法默认经期5天；若有数据可用，可灵活调整。
•	不规则周期：仅靠一次平均周期无法真正预测每个周期的波动。如用户周期极其不稳，则提示预测可能不准确；可引导用户记录更多次完整数据，用更复杂模型（如移动平均或隐马尔科夫模型）更新周期估计。
•	缺失数据：若用户提供的数据中断（如漏填经期），应推断可能的当前阶段或要求用户补充。
上述逻辑可以扩展，如支持多阶段子划分（初/晚卵泡期、前/后黄体期）或使用生物标志（体温基线、LH试纸）来精细定位排卵时点。算法的鲁棒性和准确性依赖于输入的周期数据质量，如周期规律性和记录完整性。在报告的实施阶段，将进一步讨论实现在移动端/网页端的具体技术注意事项（见后文）。

隐私与数据安全
月经周期数据属于敏感健康信息，务必严格保护。法规方面：欧盟《通用数据保护条例》（GDPR）将生理/健康数据视为特殊敏感信息，禁止未经明确同意的处理【29†L181-L189】。美国等地也有类似隐私法规（如HIPAA）。用户权利：应提供明确易懂的隐私政策，告知数据用途、存储与分享情况，不得强制捆绑。避免使用引诱式暗示（dark patterns）诱导用户泄露数据【29†L169-L178】。
技术措施：建议采取以下做法：
•	本地优先：尽可能在设备本地保存周期和症状数据，仅在必要时（如备份或同步）加密后上传。
•	加密存储与传输：服务器端数据应使用强加密，API通信使用HTTPS/TLS。
•	匿名化/脱敏：若需做数据分析，可对数据进行匿名化处理，去除可识别个人身份的信息。
•	最小化数据收集：避免收集与经期无关的敏感信息。
•	第三方合作：与数据托管或分析服务合作时，要经过严谨审查，确保合同中禁止向广告商出售健康数据。
•	用户控制：允许用户随时导出或删除其数据；对于定位或广告权限，默认关闭。
研究和媒体已报道部分经期App隐私隐患：一些应用曾将生理数据出售给广告商【29†L169-L178】。因此，本系统应承诺“不卖数据、不与无关方共享”，并建立审计机制，确保后端合规。可考虑获得相关隐私认证（如ISO27001、HIPAA合规）以增强可信度。
移动端／Web 实现注意事项
实现此系统时需兼顾移动端（iOS/Android）和Web端特性：
•	前端界面：直观的周期日历和信息卡片。各阶段信息建议以卡片或通知形式推送，界面清晰易读，语言要简洁贴心。设计中应注意色彩与图标暗示（如红色月经、橙色卵泡绿）来帮助用户快速识别当前阶段。要支持多语言切换（至少中文/英文），以及可选的字体大小调整，提高可访问性。
•	后端计算：周期算法计算量小，可本地完成。可在客户端直接计算并推送提示，也可后台定期运行同步（尤其Web端）。若需同步多设备或备份，后端应实现加密云存储。
•	健康数据集成：可考虑与手机原生健康数据平台（如Apple HealthKit、Google Fit）集成，获取用户允许的相关生物数据（体温、心率等）和同步记录。
•	通知策略：周期开始、新阶段来临或用户需关注时，通过推送通知提醒。例如，可在预期排卵前2天提醒注意排卵；在经前一周提醒PMS防护。
•	隐私合规：移动应用需遵循平台规则（iOS/Android对健康类App的隐私声明要求）和当地法律。建议在隐私政策中明确说明不收集个人身份信息，只在用户明确同意下收集周期日志和相关生理症状。
•	测试与迭代：对不同年龄、周期规律性和文化背景的用户进行实测，确保文案和算法合理。利用Beta测试收集反馈，持续完善预测模型和内容提示。
